# Homepage Refresh Design

## Goal

在不改动其他页面结构的前提下，单独重做首页，使其更像一个完成度高、真实可用的高校法学项目平台首页。

这次首页改造要同时完成两件事：

1. 先建立“平台已经可用”的第一印象。
2. 再自然承接课程体系、教学资源与项目成效。

首页不再采用均质化 section 堆叠方式，而改为更清晰的“平台门厅型首页”结构。

## Scope

本次仅调整以下范围：

- `/app/page.js`
- 首页相关组件与样式
- 首页所需的少量内容组织方式

本次不调整：

- 案例检索库页面结构
- 案例详情页结构
- 研习工作台结构
- `课程体系 / 教学资源 / 成效展示` 的页面内容

## Design Direction

整体视觉方向采用“法学专题平台气质 + 现代编辑式排版”。

要求：

- 保持高校项目平台的正式感与可信度
- 比当前版本更有首页气质与视觉记忆点
- 更像平台入口，不像说明文集合页
- 减少重复卡片和平均用力的 section 结构
- 通过更强的首屏组织和更少但更大的内容区块提升完成度

避免：

- 商业化产品首页语气
- 任务执行式说明语言
- 解释性过强、像内部策划案的文字
- 过多同构卡片导致的首页疲劳感

## Homepage Structure

### 1. Hero Hall

首页首屏采用“左文右功能面板”。

左侧承担平台身份建立：

- 平台名称
- 副标题
- 一段较短的项目说明
- 一组简洁数据

右侧承担真实入口感：

- `案例检索库`
- `研习工作台`
- `课程体系`

入口不做营销按钮，而做成正式、清楚、可点击的功能面板。

### 2. Learning Path Band

首屏下方用一段更强结构感的内容说明平台学习链路：

- 检索案例
- 阅读导读
- 进入研习

目标不是解释功能，而是让访客快速理解平台使用方式。

### 3. Real Case Preview

首页中段直接展示若干真实案例预览。

这部分要起到两个作用：

- 证明平台内容是真实存在的
- 给访客一个自然的继续浏览入口

呈现方式优先考虑更接近“阅读入口”，而不是普通列表卡片。

### 4. Courses And Resources Overview

将课程体系与教学资源压缩为一个更紧凑的概览区，而不是在首页重复展开大量说明。

这里的任务是告诉访客：

- 平台背后有八期课程体系
- 有稳定的教学资源与标准化材料支撑

它应当是概览型，不承担完整解释责任。

### 5. Impact Closing Section

首页末段只保留最有代表性的建设成效，用于收束整站认知。

它不替代独立的成效展示页，只负责在首页形成“建设已完成、成果已形成”的结束感。

## Content Rules

- 首页文案必须是面向访客的项目介绍和功能说明
- 不允许出现执行任务时的内部语言、第三者解析式语言、评审提示语、演示壳措辞
- 每一段文字都应服务于“平台介绍”而不是“解释设计意图”
- 首页整体说明量减少，入口感和浏览节奏增强

## Interaction Rules

- 首屏核心入口必须一眼可见
- 首页到案例检索、案例详情、研习工作台的链路保持清楚
- 课程体系和教学资源入口作为次一级入口出现
- 首屏数据要保留，但呈现更克制、更整洁

## Visual Rules

- 加强首屏层次、对齐关系和留白
- 增强标题区与入口区的对照关系
- 用更强的版式而不是更多装饰来提升高级感
- 可以保留现有法学/学院气质配色，但需提升首页独特性
- 首页应明显区别于普通内容页

## Verification

实现后需要至少完成以下验证：

- 首页在桌面端有明确的首屏焦点
- 首页在窄屏下仍保持入口优先级清楚
- 首页与其他页面视觉语言连续，但辨识度更高
- 文案不出现内部执行语气或解释型多余文字
- 首页能作为单独截图用于项目材料展示
