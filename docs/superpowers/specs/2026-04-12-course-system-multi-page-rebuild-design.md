# 2026-04-12 课程体系多页面重建设计

## 1. 背景

上一轮已经把课程体系从“空壳索引页”推进到“可阅读单页”，但这一版仍有两个核心问题没有解决：

- 单期课程仍然被压缩在一个详情页里，信息层级不够清晰
- 前四期与后四期材料结构差异明显，却仍被强行套进同一套正文模板

问题不在于资料不够，而在于页面结构仍然过于扁平。  
课程体系下一轮不应继续在“单页写更长”上做加法，而应把单期课程改成真正的多页面结构。

## 2. 本轮目标

本轮目标有三项：

1. 将课程体系从“八期总览 + 单期单页”改为“八期总览 + 单期主页 + 六个子页面”
2. 将八期课程的真实内容按材料类型重新组织进页面，而不是继续停留在统一空模板上
3. 先做样板期，验证结构成立后再批量扩展到其余课程

## 3. 范围

### 3.1 包含

- `/courses` 课程体系总览页的信息架构调整
- 单期课程页从单页改为“单期主页 + 六个子页面”
- 基于 `/Users/silent/Desktop/双师项目/课程资料包` 对第一期和第五期做完整样板实现
- 为后续批量扩展八期建立统一页面骨架与内容转写规则

### 3.2 不包含

- 首页、案例检索、课程视频、成效展示的结构重做
- 一次性完成八期全部多页面改造
- 将原始资料包直接公开成文件目录下载页
- 将制作过程文件原样暴露给访问者

## 4. 已确认决策

### 4.1 总体结构

- 课程体系仍然保留 `第一期` 到 `第八期` 的八期结构
- 不按新的总分类重组整站
- 每一期先进入一个 `单期主页`
- 单期主页下再拆出六个专门页面

### 4.2 单期主页

- 单期主页采用 `中等总览页`
- 既要讲清这期课讲什么，也要承担六个子页面的入口职责
- 单期主页保留顺序感
- 六个入口卡片都显示摘要预览，不做纯标题入口

### 4.3 六个子页面名称

统一采用以下命名：

1. 本期导读
2. 重点问题
3. 内容展开
4. 材料与案例
5. 学习成果
6. 教学安排

### 4.4 子页面导航

- 进入任一子页面后，顶部保留同一期六页的横向切换导航
- 不做“上一页 / 下一页”强串联
- 顺序感主要留在单期主页层

### 4.5 实施策略

- 本轮先做样板期
- 样板期确定为 `第一期 + 第五期`
- 第一期代表前四期课堂实施型材料
- 第五期代表后四期视频化/专题化材料

## 5. 页面结构重定义

### 5.1 `/courses` 总览页职责

总览页继续回答三个问题：

1. 八期课程分别在讲什么
2. 每一期进入后能看到什么结构
3. 每一期属于课程体系中的什么阶段

但总览页不再把“进入某一期”理解为进入一个完整正文页，  
而是进入该期的 `单期主页`。

### 5.2 单期主页职责

单期主页承担三件事：

1. 用较短篇幅讲清本期课程主题、位置和主要问题
2. 压缩展示本期材料线索和内容结构
3. 作为六个子页面的总入口

单期主页不是纯跳转页，也不是完整正文页。  
它是“课程说明页 + 二级导航页”的组合。

### 5.3 六个子页面职责

六个子页面各自承担单一职责，不再在一页里抢位置：

- 本期导读：讲这期课的主题、位置、切入点和阅读价值
- 重点问题：集中呈现本期真正要讨论的问题
- 内容展开：完整写出课程主线和各部分内容推进
- 材料与案例：按材料类型展示课件、导学、任务单、案例、专题阅读等
- 学习成果：展示学生报告、反馈、课堂产出等结果材料
- 教学安排：展示双师分工、课堂推进、训练重点和教学设计

## 6. 单期主页内容模型

单期主页统一包含以下内容：

1. 本期标题与阶段定位
2. 一段本期总述
3. 2 至 4 个重点问题概览
4. 一组简短材料线索
5. 六个子页面入口卡片

### 6.1 入口卡片要求

每张入口卡片至少包含：

- 页面名称
- 一句摘要
- 一条更具体的内容提示

卡片不能只写“进入本页查看”这类空话，  
而必须让用户在点击前就知道里面有什么。

## 7. 六个子页面的统一骨架

### 7.1 本期导读

内容要求：

- 说明本期主题
- 说明它处于八期课程中的哪一位置
- 说明这期课为什么值得读
- 不写空口号，不写页面说明

### 7.2 重点问题

内容要求：

- 用 3 至 6 个问题组织页面
- 问题必须具体、可读、可讨论
- 每个问题后面应给出简短解释或判断线索

### 7.3 内容展开

内容要求：

- 用若干内容块展开课程主线
- 每块都要明确“这一部分讲什么”
- 能看出从导入到分析再到收束的推进关系

这是单期课程中信息密度最高的页面。

### 7.4 材料与案例

内容要求：

- 不再做“文件名列表”
- 要按实际材料类型重写
- 每类材料都要说明它在本期课程中的作用

允许出现的内容：

- 课件主线
- 专题阅读摘要
- 法理导学要点
- 研习任务单重点
- 核心案例与补充案例

### 7.5 学习成果

内容要求：

- 轻到中等密度展示学生产出
- 重点展示学生如何回应课程问题
- 不做项目汇报式统计堆砌
- 不把原始反馈直接整页贴上去

可以采用：

- 学生报告摘要
- 课后反馈摘句
- 课堂讨论成果
- 学习任务完成方向

### 7.6 教学安排

内容要求：

- 展示双师角色分工
- 展示课堂推进方式
- 展示本期训练重点
- 展示教学设计亮点

这一页是课程支持页，不应写成行政汇报页。

## 8. 内容组织原则

### 8.1 统一骨架，不统一填法

本轮采用：

- 统一页面骨架
- 按期自材料转写内容

也就是说，八期页面长得应该属于同一套产品，  
但每一期的具体内容必须来自自身材料，而不是共用一套空泛正文。

### 8.2 前四期内容来源

前四期重点吸收：

- 课件
- PDF 讲义
- 专题阅读
- 教学材料指南
- 裁判文书法理导学
- 裁判文书研习任务单
- 学生反馈与学生报告

前四期更偏课堂实施材料，  
因此 `内容展开` 和 `材料与案例` 应该更突出课程讲授逻辑与案例讨论路径。

### 8.3 后四期内容来源

后四期重点吸收：

- 教学材料指南
- 配音稿
- manifest 结构
- 裁判文书法理导学
- 裁判文书研习任务单
- 学生反馈与学生报告

后四期虽然有制作资料，但页面主内容不能再被这些制作层文件占据。  
制作资料只允许在内部转写时作为线索，不应成为面向用户的正文标题。

## 9. 样板期策略

### 9.1 第一期样板

目的：

- 代表前四期课堂实施型材料
- 验证如何把较早期的课件、讲义、案例导学吸收到多页面结构里

重点观察：

- 单期主页是否能讲清“类案检索 -> 阅读路径 -> 争点识别 -> 表达训练”
- 材料与案例页是否能摆脱“图片佐证材料”感

### 9.2 第五期样板

目的：

- 代表后四期专题化、视频化材料
- 验证如何把教学材料指南、配音稿和 manifest 转成真正课程内容

重点观察：

- 内容展开页是否已经从“制作流程”回到“非法证据排除规则的课程主线”
- 材料与案例页是否能真正区分 `案例、规范、导学、任务、专题阅读`

## 10. 文案规范

### 10.1 禁止回退

页面中继续禁止以下表达方式：

- 本页将……
- 在这里可以……
- 用户可按顺序查看……
- 继续进入……
- 对应查看……
- 系统梳理……
- 阶段性成果……
- 多维呈现……

### 10.2 页面语言原则

- 少写页面说明
- 多写课程对象、问题、案例、规范和结论
- 少写“平台/项目/展示”
- 多写“本期课讲什么、怎样展开、学生会遇到什么问题”

## 11. 实施顺序

建议按以下顺序执行：

1. 先重构课程体系总览页的入口逻辑
2. 再实现第一期的 `单期主页 + 六个子页面`
3. 再实现第五期的 `单期主页 + 六个子页面`
4. 对第一期与第五期做文案、结构、导航联合校验
5. 确认样板后，再批量扩展第二、三、四、六、七、八期

## 12. 验收标准

本轮样板完成后，应满足以下结果：

1. `/courses` 已从“进入单个详情页”过渡到“进入单期主页”
2. 第一期与第五期均拥有 `单期主页 + 六个子页面`
3. 单期主页已展示六个入口卡片及摘要预览
4. 任一子页面顶部均可切换同一期另外五页
5. 第一期页面内容已能体现课堂实施型材料的真实结构
6. 第五期页面内容已摆脱制作层表述，回到课程内容本身
7. 整体页面语言不再出现明显说明腔、汇报腔、AI 味

## 13. 与上一版 spec 的关系

本 spec 不是对 `2026-04-12-course-archive-content-rebuild-design.md` 的简单补丁，  
而是对其单页模型的进一步重构。

保留的部分：

- 八期课程仍是主结构
- 课程页面仍然以内容优先
- 资料仍须转写，不能原样堆文件名

替换的部分：

- 单期“一页六段式”改为“单期主页 + 六个子页面”
- 统一长文页改为统一骨架下的多页面系统
- 样板期先行，而不是直接八期一起铺开
