# 2026-04-12 课程体系内容重建设计

## 1. 核心判断

当前课程体系的问题不在页面样式，也不在栏目名称，而在内容对象本身建错了。

上一轮改造把课程体系做成了“八期总览 + 单期主页 + 六个子页面”，结构比原来清楚，但页面实际消费的仍然是：

- 材料名
- 栏目壳子
- 概括句
- 局部样板文案

这会直接导致两个结果：

1. 页面看起来像有结构，实际没有课程内容
2. “学生课后反馈（一）”“学生研习报告（二）”这类原始材料标签被错当成正文

这不是页面优化问题，而是内容建模错误。  
因此，这一轮不能继续做“页面补字”，而必须做“内容重建”。

## 2. 本轮目标

本轮目标只有三项：

1. 保留现有六栏页面外壳，不重新折腾表层导航结构
2. 把课程资料包先转成“可上网站的课程内容对象”，再供页面消费
3. 优先完成 `第一期 + 第五期 + 第六期 + 第七期 + 第八期` 的完整内容重建，同时为八期建立统一标准

一句话概括：  
本轮要把课程体系从“栏目页”变成“课程内容页”。

## 3. 范围

### 3.1 包含

- 重建课程体系的内容数据模型
- 重写课程体系页面的数据来源与展示逻辑
- 基于 `/Users/silent/Desktop/双师项目/课程资料包` 对五个优先期次做完整内容转写
- 为第二、三、四期接入同一套数据结构，保留后续补实入口

### 3.2 不包含

- 改动课程体系的 URL 结构
- 重新发明新的一级栏目体系
- 把资料包原始文件公开成下载目录
- 直接展示制作清单、时长表、审核结论、API 生图文件等过程材料

## 4. 已确认决策

### 4.1 页面外壳保留

本轮保留现有六个栏目：

1. 本期导读
2. 重点问题
3. 内容展开
4. 材料与案例
5. 学习成果
6. 教学安排

不再把时间浪费在栏目名重命名上。  
这一轮先解决“里面没有内容”的问题。

### 4.2 数据层必须重建

页面不得再直接消费以下内容：

- `materialGroups` 里的原始材料名
- “学生课后反馈（一）”“学生研习报告（二）”这类标签
- 资料包制作层元数据
- 纯说明性占位文案

页面必须只消费整理后的课程内容对象。

### 4.3 前四期与后四期处理方式不同

- 前四期：以“整理转写”为主
- 第五至第八期：以“整理 + 重写 + 补做”为主

原因不是后四期没有材料，而是后四期很多材料虽然足够支撑课程重建，却并不适合直接上站。

### 4.4 第五至第八期允许合理重写

在严格贴着现有材料走的前提下，允许对第五至第八期进行合理重写和补做。

边界如下：

- 可以重写表达方式
- 可以把配音稿、教学材料指南、manifest 结构整合为网站内容
- 可以补足网站阅读所需的承接句、总结句、比较句
- 不可以脱离现有主题、案例方向、法理线索和课堂主线自由发挥

## 5. 问题拆解

当前课程体系至少有五类错误：

### 5.1 把材料标签当成正文

例如：

- 学生课后反馈（一）
- 学生研习报告（二）
- 双师合作互评问卷

这类表达只能说明“有这个材料”，不能说明“材料里有什么内容”。  
对网站访问者几乎没有阅读价值。

### 5.2 把制作层材料暴露成课程层内容

第五至第八期的大量材料属于：

- 配音稿
- 资料包制作清单
- 逐页时长表
- 时长审核结论
- API 生图结构

这些文件可以作为内容整理的线索，但不能原样上站。

### 5.3 单期主页仍然像跳转页

单期主页目前仍然偏向“目录页”，没有真正承担课程总览职责。  
访问者进入后，依然看不清：

- 本期讲什么
- 为什么从这里切入
- 核心案例是什么
- 课程主线怎么推进

### 5.4 六个栏目没有实质分工

现在的栏目虽然命名不同，但实际内容常常仍是：

- 概括句
- 标签名
- 材料名
- 空泛说明

这会导致六栏名义上分开，实质上还是空壳。

### 5.5 第五至第八期没有完成内容制作

第五至第八期当前页面更多体现为“资料改编说明页”，而不是“课程内容页”。  
它们的真实优势其实在于已经拥有：

- 教学材料指南
- 配音稿
- manifest 页结构
- 法理导学
- 研习任务单
- 学生报告与反馈

问题不是材料不足，而是没有把这些材料加工为网站内容。

## 6. 新的数据模型

课程体系不再直接由 `guide + outline + materialGroups` 驱动页面。  
这几类原始数据继续保留，但它们只作为内容生成的底层资料。

页面新增一层统一的课程内容对象：`courseContentProfile`。

每一期都必须整理出如下字段。

### 6.1 `periodSummary`

用于单期主页和本期导读页，至少包含：

- `theme`
- `position`
- `bridge`
- `lead`
- `keySignals`

含义：

- `theme`：这一期真正讨论的核心主题
- `position`：它在八期课程中的位置
- `bridge`：与前后期的衔接关系
- `lead`：一段可直接上页面的总述
- `keySignals`：2 至 4 条最重要的主线提示

### 6.2 `coreQuestions`

必须是具体问题数组。  
每个问题对象都应包含：

- `question`
- `whyItMatters`
- `judgmentPath`

要求：

- 不能只有问句
- 必须附带判断线索
- 问题要能直接拿去讨论或写报告

### 6.3 `contentFlow`

这是课程正文主线。  
每期至少 4 个内容单元，建议 4 至 6 个。

每个单元包含：

- `title`
- `goal`
- `body`
- `anchors`

含义：

- `title`：这一单元讲什么
- `goal`：这一单元解决什么问题
- `body`：这一单元的完整转写内容
- `anchors`：对应的案例、概念、规则或课堂动作

### 6.4 `caseStudy`

每期都要给出案例层内容，而不是只写“裁判文书法理导学”。

应包含：

- `mainCases`
- `supportCases`
- `controversies`
- `comparisonPoints`
- `analysisPath`

要求：

- 说明核心案例是什么
- 争点在哪里
- 为什么要比较
- 如何完成裁判分析

### 6.5 `materialsInterpretation`

材料页不再列文件。  
它要解释“材料分别承担什么课程功能”。

建议字段：

- `courseware`
- `guideReading`
- `taskDesign`
- `supportingMaterials`

每个字段都不是文件名数组，而是若干“材料功能说明对象”：

- `title`
- `role`
- `content`

例如：

- 课件主讲什么
- 法理导学抓什么问题
- 研习任务单要求学生完成什么
- 专题阅读补什么背景

### 6.6 `studentOutcomes`

这是本轮最关键的重建对象之一。  
这一页必须出现真实内容，而不是标签。

建议包含：

- `reportFindings`
- `feedbackInsights`
- `classroomOutputs`
- `learningShift`

要求：

- 学生研习报告要提炼出核心观点和典型判断
- 学生反馈要归纳出真实困惑、理解变化、收获重点
- 课堂产出要说明讨论形成了什么
- 可以出现少量摘句，但不能整页贴原文

### 6.7 `teachingDesign`

教学安排页必须从“行政表格”转成“课堂设计说明”。

建议包含：

- `mentorRoles`
- `sessionStructure`
- `trainingFocus`
- `designHighlights`

要求：

- 理论导师做什么
- 实务导师做什么
- 课堂顺序如何推进
- 每一步训练什么能力
- 为什么这样安排

## 7. 资料来源与转写规则

### 7.1 前四期资料来源优先级

1. 教学材料指南
2. 课件 / PDF 讲义
3. 裁判文书法理导学
4. 裁判文书研习任务单
5. 学生研习报告
6. 学生课后反馈
7. 双师职责分工表 / 课堂记录观察表 / 互评问卷

处理原则：

- 以整理转写为主
- 图片类佐证材料只作为内容依据，不直接作为页面正文
- 页面呈现重点是课程逻辑、案例进入和学生输出

### 7.2 第五至第八期资料来源优先级

1. 教学材料指南
2. 章节配音稿
3. manifest 结构
4. 裁判文书法理导学
5. 裁判文书研习任务单
6. 学生研习报告
7. 学生课后反馈
8. 双师职责分工表 / 课堂记录观察表 / 互评问卷

处理原则：

- 以“整理 + 重写 + 补做”为主
- 配音稿是内容主干，不是制作废料
- manifest 用于校准章节顺序和内容骨架
- 教学材料指南用于校准课程定位、学习目标、案例方向和教学亮点
- 资料包制作清单、逐页时长表、时长审核结论、API 生图信息只作为内部线索，不进入网站正文

### 7.3 第五至第八期的重写边界

允许重写，但必须满足以下四条：

1. 不改变课程主题
2. 不改变案例方向
3. 不改变法理主线
4. 不脱离已有章节结构

也就是说，可以把零散材料加工成成熟网页内容，但不能把课程改写成另一门课。

## 8. 六个栏目的新标准

### 8.1 单期主页

单期主页不再展示一串材料标签。  
它应当集中回答四个问题：

1. 这一期讲什么
2. 核心案例是什么
3. 课程主线怎么推进
4. 六个栏目分别能看到什么

必须包含：

- 一段课程总述
- 2 至 4 条主线提示
- 1 至 3 组核心案例或核心主题
- 六个栏目入口摘要

不再保留“材料线索标签墙”式呈现。

### 8.2 本期导读

必须讲清：

- 这期课的主题
- 在八期课程中的位置
- 为什么从这里切入
- 这一期值得读的原因

禁止：

- 口号式概括
- 说明页面作用的套话

### 8.3 重点问题

必须呈现：

- 3 至 6 个具体问题
- 每个问题的判断线索

禁止：

- 空问题
- 没有解释的追问

### 8.4 内容展开

这是单期最重要的正文页。  
要求：

- 以课程推进顺序完整展开
- 能看出导入、分析、比较、收束的关系
- 每一块都说明“讲什么、为什么、解决什么”

这一页必须让访问者在不看原资料包的情况下，也能知道这期课程真正讲了什么。

### 8.5 材料与案例

要求：

- 写材料功能，不写文件清单
- 写案例争点，不写材料目录
- 写比较路径，不写“有这个材料”

禁止出现：

- 原始文件名列表
- 资料包制作层文件
- “学生课后反馈（一）”这类纯标签

### 8.6 学习成果

这是最不能再糊弄的一页。  
要求：

- 必须有真实内容
- 必须提炼学生观点、困惑、理解转变和课堂产出
- 必须能看出学生对课程问题作出了什么回应

禁止：

- 只挂“学生研习报告”“学生课后反馈”字样
- 只写有多少项材料
- 只堆空泛总结句

### 8.7 教学安排

要求：

- 把双师分工讲清楚
- 把课堂推进讲清楚
- 把训练重点讲清楚
- 把为什么这样设计讲清楚

禁止：

- 行政报表转写
- 只有结构名，没有课堂内容

## 9. 本轮优先级

### 9.1 本轮完整重建的期次

- 第一期
- 第五期
- 第六期
- 第七期
- 第八期

理由：

- 第一期代表前四期课堂实施型材料
- 第五至第八期是当前最失真的部分，必须优先纠正

### 9.2 本轮接入统一骨架但暂不补满的期次

- 第二期
- 第三期
- 第四期

要求：

- 接入新的 `courseContentProfile` 结构
- 去掉明显错误的标签式呈现
- 为下一轮补实保留统一入口

## 10. 技术落地方向

本轮不推翻现有路由层，但要重做 `lib/course-system-pages.js` 的职责。

它不再负责：

- 手工拼接样板概括文案
- 用 fallback 文案填满页面

它要负责：

- 从原始课程数据生成 `courseContentProfile`
- 让页面只消费整理后的内容对象
- 区分“原始材料元数据”和“网站可读内容”

必要时可以新增专门的内容构建文件，例如：

- `lib/course-content-profiles.js`
- `lib/course-content-transforms.js`

页面组件层的目标很简单：

- 不再直接读材料名
- 不再负责内容推断
- 只负责渲染结构化内容

## 11. 验收标准

这轮是否合格，只看以下标准：

1. 页面中不再出现原始材料名充当正文
2. 第五至第八期能明显看出经过内容制作，而不是资料拼装
3. 学习成果页出现真实内容，而不是标签或清单
4. 内容展开页能独立成立，不需要用户反推资料包
5. 单期主页能清楚展示本期主题、核心案例、主线和栏目入口
6. 第一、第五、第六、第七、第八期进入后，都能读到具体课程内容

## 12. 非目标提醒

本轮不是：

- 修饰语气
- 再做一轮表层美化
- 再补几段解释性文案

本轮是一次内容层重建。  
如果内容对象不重建，课程体系会持续停留在“看起来有页、实际上没内容”的状态里。
