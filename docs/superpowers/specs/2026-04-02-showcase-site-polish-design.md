# 裁判文书研习平台网站深度打磨设计稿

## 1. 任务定义

本轮工作的目标，不是继续做零散的文案替换，而是在现有独立展示站基础上进行一轮更接近成品级别的网站打磨。打磨范围同时覆盖：

- 首页的正式感、完成度与入口组织
- 案例检索库、案例详情页、研习工作台的页面成立度
- 课程体系、教学资源、成效展示三页的统一感
- 全站文案口吻、信息密度、留白节奏与视觉秩序

本轮的核心问题已经明确：案例研习页面缺少相应案件 PDF 文件，导致阅读链路不完整，直接拉低了整站的可信度和完成度。

## 2. 本轮设计目标

### 2.1 总体目标

将现有展示站从“已经可用的申报站”提升为“具备高校专题平台成品感的法学教学成果网站”。

目标状态包括：

- 页面口吻稳定，完全站在访客视角
- 关键页面结构成立，不因局部资源缺失而显得半成品
- 首页、案例链路和研习链路具有足够强的截图说服力
- 非重点页与重点页之间风格统一，不出现明显断层

### 2.2 优先级

本轮采用“重点页拉高、全站同步提升”的策略。

重点页：

1. 首页
2. 案例检索库
3. 案例详情页
4. 研习工作台

同步提升页：

1. 课程体系
2. 教学资源
3. 成效展示

## 3. 设计原则

### 3.1 气质原则

整站视觉与文案气质采用：

- 以学院感、正式感、可信度为基底
- 吸收成熟教育产品的秩序感与完成度
- 避免商业化互联网产品风
- 避免老式信息门户或政务站式陈列

### 3.2 文案原则

文案只保留真正面向访客的项目介绍、功能说明与阅读引导，不出现以下表达：

- 任务执行痕迹
- 设计者自我解释
- “公开展示模式”“本页用于”“截图”“评审”等内部任务口径
- 为实现妥协而产生的防御性说明

页面文案应满足：

- 能介绍项目
- 能引导浏览
- 能支撑教学成果语境
- 不显得像项目组内部备忘录

### 3.3 页面成立原则

每个页面都必须在自身成立，而不是依赖访客脑补。

具体要求：

- 首页不能像功能拼盘
- 案例详情页不能只是数据堆叠
- 研习页不能因为缺 PDF 而像残缺页面
- 成效页不能只是数字罗列

## 4. PDF 资源策略

### 4.1 总策略

优先从原项目中迁移已有的真实 PDF 文件，已有多少迁多少；暂时没有的允许留空，但页面必须成立。

### 4.2 不追求的目标

本轮不把“补齐全部案例 PDF”设为前置条件，不因个别缺失资源阻塞整站打磨。

### 4.3 页面层面的承接方式

有 PDF 的案例：

- 保留阅读区
- 强化“原文阅读 + 研习输出”的完整链路

无 PDF 的案例：

- 不再硬撑大面积空阅读器
- 主动切换成以导读、事实摘要、争点提示和研习输出为主的页面结构
- 让访客感受到这是“另一种成立的阅读路径”，而不是“缺文件后的退化状态”

## 5. 关键页面方案

### 5.1 首页

#### 定位

首页需要同时完成两件事：

1. 建立“法理学教学改革成果平台”的正式身份
2. 让访客一眼看见“案例检索库”和“研习工作台”是真实可进入的

#### 首屏结构

首屏采用折中型结构：

- 左侧主叙事
  - 平台名称
  - 副标题
  - 一段简洁项目说明
- 右侧主入口
  - 进入案例检索库
  - 进入研习工作台
  - 可补一个课程体系或教学资源入口
- 下方关键数据
  - 210+ 典型案例
  - 8期 双师课程
  - 5万+ 累计访问

#### 首页打磨重点

- 减少“解释平台在做什么”的冗余文案
- 强化区块之间的层次感
- 提高首屏和第二屏之间的节奏落差
- 让首页从“专题说明页”更接近“正式平台首页”

### 5.2 案例检索库

#### 定位

案例检索库要证明平台不是静态展板，而是真实可用的案例组织界面。

#### 打磨方向

- 强化筛选区、结果区、辅助提示区之间的主次关系
- 让检索页更像“工作区”，而不是一页功能说明
- 对搜索提示、筛选提示和结果说明进行收束
- 提升顶部区域与结果列表之间的呼吸感和秩序感

#### 目标效果

访客进入后能立即理解：

- 如何缩小范围
- 当前结果是否过多
- 该从哪一类案例先进入详情页

### 5.3 案例详情页

#### 定位

案例详情页不是资料堆叠页，而是“案例导读判断页”。

它要帮助访客回答：

- 这份案例讲什么
- 核心争点在哪里
- 值不值得继续进入研习

#### 结构重点

- 顶部基础信息
  - 案名、案号、法院、日期、裁判结论
- 案例导读
  - 收束案件事实、核心争点、裁判方向
- 继续阅读依据
  - 文书入口、关联法条、适合进入研习的角度
- 进入研习工作台的明确动作

#### 打磨目标

- 从“信息页”转向“导读页”
- 提高摘要、法条、裁判结果之间的连贯性
- 让“进入研习”成为自然动作，而不是页面末端附加按钮

### 5.4 研习工作台

#### 定位

研习工作台是整站最关键的教学落地点，要直接证明教学法已经平台化落地。

#### 采用方案

采用“条件切换型”布局。

#### A. 有 PDF 时

保留真正的双栏工作台：

- 左侧原文阅读区
- 右侧结构化研习输出区

目标是明确呈现“文书阅读 + 三步研习”的完整工作流。

#### B. 无 PDF 时

不再保留空阅读区，而改为：

- 顶部案件导读区
- 下方结构化研习工作台

顶部导读区可整合：

- 案件摘要
- 关键事实
- 争点提示
- 法条与裁判方向

这样页面在没有 PDF 的情况下依然成立，并且更贴合教学中的“导学后进入研习”逻辑。

#### 研习页打磨重点

- 去掉任何“因为缺资源所以这样处理”的姿态
- 把无 PDF 状态设计成正常路径，而不是替代方案
- 强化研习步骤、参考要点、填写区、结尾动作之间的纵向节奏
- 让页面从“工具页面”提升为“学习工作台页面”

## 6. 其他三页的同步提升

### 6.1 课程体系

保持“时间轴 + 课程结构”的核心逻辑，但强化：

- 标题与引导段的正式感
- 每一期课程卡片的层次
- 时间轴与总结模块之间的节奏切换

### 6.2 教学资源

保持“资源成果说明为主、资源分类为辅”的结构，但强化：

- 资源体系的整体性
- 标准化材料的可复制感
- 示例资源的版面完整度

### 6.3 成效展示

保持“成果价值分块”为主的结构，但强化：

- 板块之间的差异化层次
- 数据与价值表述的平衡
- 平台运行成效与教学建设成效的前后呼应

## 7. 实施边界

### 7.1 本轮包含

- PDF 资源迁移与接入
- 关键页结构重组
- 无 PDF 研习页布局切换
- 文案继续收束
- 排版、留白、层级和视觉秩序优化

### 7.2 本轮不包含

- 新增登录体系
- 恢复完整后台能力
- 大规模新增案例内容生产
- 对原始案例数据做重写式整理

## 8. 验收标准

完成后应满足：

1. 首页单独截图即可体现正式项目平台气质
2. 检索库、详情页、研习页形成清晰连续链路
3. 有 PDF 的案例能体现真实阅读场景
4. 无 PDF 的案例不会显得页面残缺
5. 全站不再出现任务执行式或第三者解析式文案
6. 重点页与非重点页之间不出现明显成品感断层

## 9. 推荐实施顺序

1. 清点并迁移原项目中的可用 PDF
2. 改造研习工作台为“双态布局”
3. 重构案例详情页的导读逻辑
4. 提升首页首屏与整体完成度
5. 同步优化检索库与三个内容页
6. 做最终截图页抽查与视觉回归
