# 课程体系导师与第一期视频补齐 Design

## 背景

上一轮课程体系重构已经完成单期课程入口、统一材料框架和八期课程档案的主结构，但课程体系仍有三处明显缺口：

1. 第一至第四期导师信息不统一，部分期次缺少导师单位，第二、三、四期在部分页面甚至只显示姓名。
2. 第一期视频只保留了“第一期上”四段，缺失“第一期下”对应的站内入口，导致第一期视频成果链路不完整。
3. `/courses` 页面中“统一材料框架”和“课程档案”两个板块上下距离过近，页面呼吸感不足。

这三处问题都发生在专家直接可见的公开页面，优先级高于任何新增装饰或结构性改版。

## 本轮最高原则

1. 所有公开页面中的导师信息必须完整到“姓名 + 单位”粒度，且八期统一。
2. 导师补齐必须从数据源层处理，而不是在单个页面做展示层补丁。
3. 第一期视频补齐必须是具体内容补齐，不允许写成“待补充”“补录中”“下半场入口”等过程性占位话术。
4. 第一期下视频文案必须基于真实视频内容生成，禁止照抄第一期上的四段标题，也禁止写空泛总结。
5. `/courses` 间距修正只做页面定点优化，不能通过放大全局 `.showcase-section` 间距引入其他页面回归。

## 设计决策

### 1. 导师信息在源头标准化

在 `lib/course-package.js` 增加导师单位补齐逻辑，原则如下：

- 已从 HTML 指南读出的导师字段优先使用。
- 若 HTML 指南只给出姓名或“律师/博士/仲裁员”等身份描述，则用本轮确认的单位表补齐为完整字符串。
- 补齐结果统一写回 `guide.theoryMentor` / `guide.practiceMentor`，让课程首页、单期首页、材料页、课程档案页沿同一数据流自动生效。

这样可以避免在 `course-period-shell`、`course-archive-content`、`course-material-pages` 等多个下游文件重复拼接单位。

### 2. 第一期视频入口补成“完整五段”

第一期视频仍维持 `segments` 模式，不改播放器结构。

本轮只补一个新的分段卡片：

- 标签：`第一期下`
- 标题：围绕“类案检索意义 + 优先承租权案细读”组织
- 说明：明确该段从类案检索的制度价值转入具体判决书展开

原因：

- 当前真实可核实资源只有一个“第一期下”独立视频链接。
- 现有资源中心页已经支持“若干分段卡片 + 外链跳转”，补一张新卡即可闭合链路。
- 如果后续拿到更细的下半段拆分资源，再扩成多段，不影响当前数据结构。

### 3. 课程体系页只做定点间距优化

`/courses` 的问题不是卡片内部样式，而是两个 section 紧贴。处理方式：

- 在 `app/courses/page.js` 为第二个 section 或父级容器增加更明确的页面级 class。
- 在 `app/globals.css` 中新增页面专用的纵向间距规则，只调整 `统一材料框架 -> 课程档案` 这组相邻 section。

不修改全局 section 默认间距，避免影响首页、资源页、成效页和单期材料页。

## 非目标

- 本轮不重做课程体系首页整体 UI。
- 本轮不改单期课程首页正文结构。
- 本轮不继续拆分或重录第一期视频。
- 本轮不处理 Cloudflare / GitHub 发布，除非实现完成后用户再次要求。
